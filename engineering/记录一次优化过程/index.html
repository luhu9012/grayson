webpack 构建优化

  1. 安装 speed-measure-webpack-plugin、webpack-bundle-analyzer

    1. 构建速度测量： @vue/cli 4.5.9 vue.config.js中应用插件

    2. 构建包大小分析：查看是否已安装webpack-bundle-analyzervue inspect plugins，安装webpack-bundle-analyzer vue add webpack-bundle-analyzer，运行 yarn build –report 或者 npm run build – –report

    3. 优化前：
      构建时间：1 min, 5.49 secs

    输出文件大小：

优化方案1：缩小文件搜索范围

  • 优化 loader 配置：test 、 include 、 exclude
  • 优化 resolve.modules 配置

  • 优化 resolve.mainFields 配置: mainFields: [‘main’]

  • 优化 resolve.alias 配置

  • 优化 resolve.extensions 配置

  • config.module.noParse

    优化结果差异：65.49减少到 54.37 secs，速度提升16.9%

优化方案2：压缩图片 安装image-webpack-loader插件

优化前后对比：图片136326减少到58399，减少57.16%体积：


优化方案3：使用 DllPlugin、DllReferencePlugin 或者使用autodll-webpack-plugin替代前两者

  • DllPlugin 插件：用于打包出一个个单独的动态链接库文件。
  • DllReferencePlugin 插件：用于在主要配置文件中去引入 DllPlugin 插件打包好的动态链接库文件。

步骤：

  • 新建webpack.dll.config.js文件(其他命名都可以)，配置需要拆分的插件；

  • 在package.json文件中新建一条命令来专门打包，”build:dll”:”webpack –config webpack.dll.config.js”; 运行该命令；

  • 在vue.config.js 文件中配置DllReferencePlugin,主要把dll引用到需要预编译的依赖；

  • 在index.html手动引入拆分的bundle包（放到cdn的话会更好）

    优化结果差异： 54.37 减少到 21.24 sec ，速度提高60%

    ！！！vue-cli 与 create-react-app 都不用dll了，因为webpack 4 有着比 dll 更好的打包性能

    比Dll方案更优秀的插件HardSourceWebpackPlugin，该配置已经默认包含在 webpack 5 里，开箱即用

    使用HardSourceWebpackPlugin：优化前后对比：52.99(第一次构建) 减少到 13.71 secs(第二次build), 本地开发构建速度提升75% (build) ,第一次serve took 37.2 secs,第二次 serve took 2.84 secs, 速度提升92.36%

优化方案4： cache-loader vue-cli@4已经默认给babel loader前加了cache-loader，但是scss-loader没有加cache-loader，而scss-loader又是构建过程中最耗时的loader

优化方案5：CICD中使用npm ci代替npm i， 消耗时间从228.108s增加到246.565s ！！！！
